ARCHITECTURE & UI/UX SPECIFICATION: NEXT-GEN OPEN SOURCE CHAT APP1. VISION & DESIGN PRINCIPLESTarget: Build an open-source chat application (Discord/Slack hybrid) focused on ultimate simplicity, performance, and modern developer aesthetics (Linear/Telegram-like).Anti-Feature Creep: Eliminate unnecessary multi-column clutter. Maximize screen real estate for actual content.Design Tokens:Theme: Modern Dark/Light Mode (Zinc/Slate palettes from shadcn/ui).Radius: Large rounding (radius-xl / 12px to 16px) for cards, avatars, and floating containers to create a soft, friendly, and premium tech feeling.Typography: High-readability sans-serif font system with generous line-height (leading-relaxed).2. LAYOUT STRUCTURAL UPGRADES (THE ANTI-CLUTTER SIDEBAR)Old Problems:Discord: Infinite stacked circular server icons on the far left + long channel lists = visual overload.Slack: High cognitive load when switching workspaces (requires opening separate tabs or instances).Next-Gen Solution: 2-Stage Collapsible Dynamic SidebarStage 1: Ultra-Slim Navigation Bar (Width: 64px - Fixed)Top: Global workspace active indicator (Clean typography instead of noisy server avatars).Center: Vertical Context Filters (Icons only): [All messages], [Work Spaces], [Social Communities], [Direct Messages (DMs)].Bottom: User Profile Trigger & Quick Settings gear.Stage 2: Fluid Content Explorer (Width: 240px - Collapsible)Contains the nested structure of the active filter.Uses a Folder-Tree Paradigm: Spaces can be grouped into folders. Channels (Text #, Voice 🔊) live inside spaces.UX Improvement: A dedicated keyboard shortcut (Cmd+B / Ctrl+B) instantly hides this Explorer stage, expanding the chat viewport to maximum size.3. CORE COMMUNICATION PROTOCOLS (UI/UX SPECIFICATION)A. Direct Messages (1:1 Chat) & Private Group ChatsPresence Indicator: Instead of a simple green dot, use an ambient glow border around the user avatar to signify "Active".Contextual Dynamic Input Bar:The message box automatically shows relevant actions based on the chat type.For 1:1 chats, a ghost icon for "Quick Actions" (e.g., /pay, /schedule, /share-location) appears on the right edge.Group Management: Creating a group chat does not redirect users to a heavy settings screen. It triggers a lightweight modal with an instant fuzzy-search input to tag and add members, immediately spinning up the chat room via a single gRPC event.B. Public Channels & The "Hybrid Inline Thread" SystemOld Problem: Slack's secondary right-sidebar threads feel cramped and split user attention. Discord's flat timeline causes messages to drown instantly.Next-Gen Solution: Inline Expandable Threads (Accordion Style)When a user replies to a specific message, a small, elegant branch line extends down from that message.The main chat view displays the first 2 replies inline. A pill button states [ View 5 more replies ].Clicking this button gracefully expands the thread directly inside the main timeline without pushing the layout sideways.Users can easily follow separate micro-conversations without losing the global channel context.C. Voice Channels & The "Dynamic Island" Call InterfaceOld Problem: Joining a voice channel in Discord requires hunting down the exact channel in the sidebar. Moving away from that view hides call controls.Next-Gen Solution: Frictionless Voice & Floating Control BubbleSingle-Click Join: Clicking a voice channel 🔊 Room Alpha instantly connects the user without any loading screen/pop-up.The Persistent Dynamic Widget: When the user navigates away to a completely different space or text channel, a small rounded horizontal pill (inspired by Apple's Dynamic Island) anchors smoothly at the top-center of the main chat area.Control States: This floating pill displays active speakers via small bouncing waveforms, and exposes instant-access toggle triggers for Mute, Deafen, Screen Share, and Leave. Hovering expands it to show detailed participant avatars; clicking returns the user to the original voice room view.D. Direct & Group Calls (Audio & Video)Incoming Call UX: Replaces the disruptive full-screen ringing takeover with an elegant, sliding notification banner from the top right. It gives clear Accept / Decline controls with an ambient sound wave animation.Grid Allocation: Video grids dynamically adapt (16:9 masonry layout). The current speaker's grid box gets a subtle, color-shifting border transition.Mini-Player Mode: Dragging or clicking away during a live video call minimizes the call into a high-frame-rate, picture-in-picture floating overlay component in the bottom-right viewport corner.4. CONTEXTUAL & ASYNC DATA MANAGEMENTA. Media Uploads & Drag-and-Drop PipelinesThe Drag-Overlay Experience: Dragging any file type anywhere over the viewport darkens the entire interface smoothly, rendering a dotted bounding frame stating: Drop to instant upload via SeaweedFS.Inline Upload Micro-Component:Dropped files are captured by the Frontend. It fetches a Presigned URL instantly from the Go backend.The file does not block the chat input. Instead, the input returns to an empty state immediately, and an un-sent message template appears in the message history list containing a loading progress spinner (0% -> 100%).Once the background upload to the object storage completes, the component automatically transforms into the finalized rich message format (Image preview grid, file attachments card, or markdown snippet).B. The Omni-Command Palette (Cmd+K / Ctrl+K)The primary interaction point for power users. A centered, blur-backed modal overlay inspired by Linear.app.Instant Capabilities:Type t:[space/channel name] $\rightarrow$ Fuzzy find and jump to any text channel.Type v:[room name] $\rightarrow$ Instantly join a voice room.Type /call @username $\rightarrow$ Trigger an instant direct call event.Type /away or /busy $\rightarrow$ Mutate personal presence state instantly.5. DESIGN & FRONTEND IMPLEMENTATION GUIDELINES (FOR THE AI CODE)Component Strategy: Utilize standard primitive elements from Radix UI wrapped with Tailwind CSS v4 styling properties. Keep components highly decoupled (Separate MessageView from InputController).Performance Constraint: Enforce strict window virtualization via @tanstack/react-virtual for all scroll regions (Channels, DMs, Threads) to enforce stable 60 FPS scrolling independent of chat database size.Real-Time State Mapping: Network events ingested through the WebSocket protocol must write directly into the localized caching layer (TanStack Query). Component layers must derive state through this hook ecosystem to prevent manual re-renders.
